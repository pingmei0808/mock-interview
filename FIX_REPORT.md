# 面试模拟项目 - 追问逻辑修复报告

**修复日期：** 2026-03-27  
**问题：** 面试官重复问问题，无法基于用户回答进行追问  
**状态：** ✅ 已修复

---

## 🔍 问题分析

### 原代码核心问题

1. **Prompt 设计缺陷**
   - 没有明确区分"新问题"和"追问"模式
   - 模型容易忽略"基于回答追问"的指令
   - 缺少已问问题追踪机制

2. **状态管理混乱**
   - `currentRound++` 在每次生成问题时都增加，而非用户回答后
   - 没有追问深度计数器
   - 无法控制每轮的追问次数

3. **API 配置缺失**
   - 没有 API Key 配置界面
   - 前端直接调用需要用户配置 Key

---

## ✅ 修复方案

### 1. 新增状态变量

```javascript
state = {
    currentRound: 1,           // 当前轮次（从 1 开始）
    totalRounds: 5,            // 总轮数
    followUpDepth: 0,          // 追问深度：0=主问题，1~max=追问
    maxFollowUps: 2,           // 每轮最大追问次数
    askedQuestions: [],        // 已问问题列表（避免重复）
    apiKey: ''                 // 用户 API Key
}
```

### 2. 追问逻辑流程

```
开始面试
    ↓
第 1 轮 - 主问题 (followUpDepth=0)
    ↓
用户回答
    ↓
追问 1 (followUpDepth=1) ←─┐
    ↓                      │
用户回答                   │
    ↓                      │
追问 2 (followUpDepth=2) ←─┘
    ↓
进入第 2 轮 (followUpDepth=0, currentRound=2)
    ↓
... 重复直到所有轮次完成
    ↓
生成面试反馈
```

### 3. Prompt 优化

**追问模式：**
```
【追问模式】这是第 X 次追问，请基于候选人刚才的回答进行深入追问。
追问要点：
- 针对候选人回答中的具体细节或模糊点
- 询问"为什么"、"如何做"、"结果怎样"等
- 不要重复之前已经问过的问题
- 保持问题简洁，一次只问一个点
```

**新问题模式：**
```
【新问题模式】请开启一个新的面试问题，要求：
- 基于 JD 中的核心能力要求
- 避免与已问问题重复（见下方已问问题列表）
- 问题要具体、有场景感
```

### 4. 已问问题追踪

```javascript
// 在 prompt 中注入已问问题列表
# 已问过的问题（不要重复！）
1. 请先做自我介绍
2. 为什么选择我们公司？
3. ...
```

### 5. API Key 配置

- 新增 API Key 输入框（密码类型）
- Key 存储在 localStorage，仅本地使用
- 提供 Anthropic 官方 Key 获取链接

## 🚀 使用说明

### 首次使用
1. 打开 `index.html`
2. 填入 Anthropic API Key（[获取链接](https://console.anthropic.com/settings/keys)）
3. 填写岗位信息和简历
4. 选择面试类型、轮数、追问次数
5. 点击"开始面试"

### 面试中
- 每轮包含：1 个主问题 + N 次追问（N 由用户设置）
- 面试官会基于你的回答进行追问
- 所有问题不会重复

### 面试后
- 自动生成分数、优点、缺点、建议
- 点击"再来一次"可重新开始
